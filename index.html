الدورات

مقدمة إلى أمن المعلومات

المفاهيم الأساسية: مفاهيم السرية، السلامة، التوافر، ونماذج التهديد.

الشبكات وآمنها

مفاهيم الشبكات، تحليل حزم (نظري)، وبروتوكولات آمنة.

تأمين الويب (OWASP)

مقدمة عملية ونظرية لمخاطر تطبيقات الويب وكيفية الحماية منها.

المختبرات القانونية

تدريبات داخل بيئة محلية أو منصات معروفة — لا تقم بتجربة أي هجوم على أنظمة ليس لديك إذنها.

مختبر التعلم الآمن

كيفية إنشاء بيئة افتراضية (VM) واستخدامها لتجارب آمنة — أدوات مقترحة: VirtualBox, Vagrant.

منصات تدريب شرعية

تعرّف إلى منصات مثل TryHackMe و HackTheBox لتطبيق مهاراتك ضمن قواعد قانونية.

تحليل الشيفرة والبحث عن ثغرات

مناهج لتحليل الكود بنّاءً: مراجعات كود آمنة، واختبارات وحدات لتقليل الأخطاء.

أمثلة كود آمنة (مع شرح)

// مثال: منع XSS (جافاسكربت - عرض آمن)
// لا تستخدم innerHTML مباشرة على مُدخلات المستخدم.
// بدلًا من ذلك ضع النص عبر textContent أو عن طريق إنشاء عناصر.

const userInput = '';
const container = document.createElement('div');
container.textContent = userInput; // تَحول أي وسوم لترميز نصي
document.body.appendChild(container);
ما الذي يفعل هذا الكود؟

يوضّح كيف نعرض نص مستخدم بأمان دون تنفيذ وسوم HTML أو سكربتات — نستخدم textContent بدل innerHTML.

// مثال: استعلام مُعَلَّم (Prepared Statement) — وقائي ضد SQL Injection
// هذا كود توضيحي (شبه-صِيغي) لقاعدة بيانات في back-end.

const sql = "SELECT * FROM users WHERE email = ?";
// ثم تمرير قيمة البريد كمعامل Prepared parameter
db.query(sql, [email], (err, rows) => {
  // المعاملات تُعامل بأمان من دون دمج النصوص داخل الاستعلام
});
ما الذي يفعل هذا الكود؟

يوضح مفهوم استخدام استعلامات مُعلّمة لعدم السماح بتعديل بنية الاستعلام عبر مدخلات المستخدم، وهو أسلوبٌ أساسيٌ للحماية من SQL Injection.

// مثال: تشفير كلمات السر (مبدأ) - Node.js (مكتبة bcrypt)
// هذا مثال آمن يوضح تخزين كلمة مرور مُشفَّرة في الخادم

const bcrypt = require('bcrypt');
const plain = 'user-password';
const saltRounds = 12;
bcrypt.hash(plain, saltRounds).then(hash => {
  // خزّن hash في قاعدة البيانات بدلاً من النص الصريح
});
ما الذي يفعل هذا الكود؟

يوضح فكرة عمل التخزين الصحيح لكلمات السر — استخدام دوال تجزئة قوية (bcrypt) بدلاً من تخزين كلمة السر النصية.

// مثال: التحقق من صحة المدخلات (Client-side) - بسيط وآمن
function isEmail(value){
  return /^[^\\s@]+@[^\\s@]+\\.[^\\s@]+$/.test(value);
}
ما الذي يفعل هذا الكود؟

يتحقق من صيغة البريد إلكتروني على جانب العميل. لكن التحقق الحقيقي والنهائي يجب أن يكون على الخادم أيضًا.
